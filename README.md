# D&D 5e Studio

Local, framework-free character editor and generator for Dungeons & Dragons 5e (2014 and 2024 editions) with data sourced from the 5etools mirror repositories. The app runs entirely locally on a Node.js server, at its core without npm dependencies; the only third-party library is a bundled pdf-lib for the sheet export.

This document is the developer documentation: architecture, function reference of the modules, data pipeline, and extension points.

## Quick Start

```bash
node server.js          # starts on http://localhost:8420
node tools/update.js    # optional: load full data from GitHub
node tools/test.mjs     # functional test suite (125 assertions) against the packs
```

Windows alternatives in the project folder: `DnD5e-Studio.exe` starts the server and browser and is guaranteed to terminate the Node process when the window is closed (job object with KILL_ON_JOB_CLOSE, source code `tools/launcher.c`). `Start.vbs` starts invisibly, `Start.bat` visibly with logs, `Stop.vbs` terminates running servers.

Without an update, the app runs with SRD seed data. `GET /api/update` (or the button in the settings) loads all books, classes, races, spells, items, backgrounds, beast forms, and feats and stores them as packs under `public/data/packs/`.

## Architecture and Data Flow

The data flow is unidirectional: inputs in the panels call `store.update(patch)`, the store persists and publishes `CHAR_CHANGED` with the list of changed fields, and each panel only re-renders when one of its relevant fields (its own `rel` list) is affected. Rules knowledge lives exclusively in `src/rules/` (pure functions without DOM), data access exclusively in the `DataRepository`.

### EventBus (`core/EventBus.js`)

`bus.on(event, cb)` registers (returns an unsubscribe function), `bus.emit(event, payload)` publishes. Events in `EV`:

| Event | Meaning |
|---|---|
| `CHAR_CHANGED` | Character field changed; payload `{ changed: [fieldnames] }`, `'*'` stands for everything |
| `CHAR_LOADED` | Different character active (switch, new, import) |
| `DATA_READY` | Packs or seed loaded, libraries usable |
| `LANG_CHANGED` | Language switched |
| `SOURCES_CHANGED` | Source books or ruleset switched |
| `ROLL_RESULT` | Roll result for the result display |
| `TOAST` | Show a short message |

### Store (`core/Store.js`)

Manages the multi-character roster in `localStorage` and the active character state. New characters are only written to the roster on the first substantive change (prevents empty orphan entries).

| Function | Description |
|---|---|
| `blankCharacter()` | Complete default schema; every schema extension is entered here with a default, `replace()` and `loadCharacter()` migrate legacy data through it |
| `get()` / `field(name)` | Read state (copy or single field) |
| `update(patch)` | Set fields, persist, publish `CHAR_CHANGED` |
| `quietUpdate(patch)` | Like `update`, but without an event (preserves focus during text input) |
| `updateClass(idx, patch)` / `addClass()` / `removeClass(idx)` | Maintain the class list |
| `totalLevel()` | Sum of all class levels |
| `newCharacter(ruleset)` | Fresh character; also stores the ruleset globally |
| `loadCharacter(id)` / `deleteCharacter(id)` / `listCharacters()` / `activeId()` | Roster management; loading publishes `CHAR_LOADED` and synchronizes the ruleset |
| `replace(character)` | Replace the entire character (generator, import), with schema migration |
| `exportJson()` / `importJson(json)` | Single character as JSON |
| `longRest()` | Full HP, half the hit dice back, all slots, pact slots, arcana, wild shape |
| `shortRest()` | Ruleset-dependent: pact slots always; wild shape 2014 fully, 2024 plus one use (maximum 2); arcana remain spent |
| `undo()` | Revert the last change |

Important schema fields: `ruleset` (`'phb14' | 'phb24'`), `classes` (`[{name, level, subclass}]`), `feats` parallel to `featLevels` (level of the choice; duplicates allowed for repeatable feats), `raceBonus/bgBonus/featBonus` (additive ability bonuses) with `raceChoice/bgChoice` (chosen distribution of freely assignable bonuses), `arcanumUsed` (spent Mystic Arcana), `acManual` (suppresses the automatic AC calculation), `wildshape` (active beast form).

### DataRepository (`core/DataRepository.js`)

Singleton `repo`. `load()` loads packs with seed fallback and publishes `DATA_READY`.

| Function | Description |
|---|---|
| `bySource(list)` | Filters by enabled source books |
| `dedupeByName(list)` | Ruleset dedupe: precedence homebrew, then the edition family of the active ruleset (2014: PHB/MM/DMG, 2024: XPHB/XMM/XDMG), then the other family; automatically falls back to the other edition if a book is disabled |
| `getClasses()` / `getRaces()` / `getBackgrounds()` / `getSpells()` / `getFeats()` / `getBeasts()` / `getItems()` | Filtered (and, except for items, deduplicated) views |
| `getClass(name)` | Class in the version of the active edition, subclasses also deduplicated; fallback to the unfiltered list so that saved characters can resolve their class even when the source is disabled |
| `findSpell/findItem/findFeat/findBeast(name)` | Name resolution, edition-aware with fallback cascade |
| `findSpellCI(name)` | Case-insensitive spell lookup (subclass data is lowercase) |
| `getSpellsForClasses(classes, featNames)` | Selectable spells: class lists, exclusive subclass spells (`spells.extra`), feat-granted spells (concrete names and criteria `{levels, schools, classNames}`), homebrew |
| `subclassAutoSpells(className, subclassName)` | Automatically prepared subclass spells as `{classLevel: [names]}` |
| `addHomebrew(kind, entry)` / `removeHomebrew(kind, name)` / `getHomebrew()` | Homebrew, bound to the active character (`charId`); legacy entries without binding remain global; `refreshHomebrew()` rebuilds the view on character switch |
| `setRuleset(rs)` / `setSourceEnabled(src, on)` / `isSourceEnabled(src)` / `allSources()` / `setAdvanced(on)` | Ruleset and source control, publish `SOURCES_CHANGED` |

### i18n and Theme

`t('block.key')` returns the text of the active language (de/en, `setLang`/`getLang`, `LANGS` as `[{id, label}]`). Game terms such as spell, item, and language names remain English, consistent with the data and the PDF sheet. `theme.js` manages six warm color presets with separate light and dark variants (all 24 combinations numerically checked against WCAG AA); `applyTheme` resolves the effective mode (for "System" with live switching via matchMedia) and sets the variant as inline variables.

## Function Reference: Rules Logic (`src/rules/`)

### calculations.js

| Function | Description |
|---|---|
| `calcMod(score)` | Ability modifier, `floor((score - 10) / 2)` |
| `calcProfBonus(totalLevel)` | Proficiency bonus 2 to 6 |
| `fmtMod(n)` | Sign format (`+3`, `-1`, `+0`) |
| `calcSkillBonus(mod, prof, expertise, pb)` | Skill bonus, expertise doubles the proficiency bonus |
| `SKILL_DEFS` / `ABILITY_IDS` | 18 skills with ability mapping; ability IDs |
| `effectiveAbilities(state)` | Effective values `{scores, sources}` from base plus race, background, feat bonuses and item fixed values; in wild shape the beast form's physical scores replace STR/DEX/CON, mental scores remain |
| `effMod(state, abilityId)` | Shorthand for `calcMod(effectiveAbilities(...).scores[a])` |
| `calcMaxHP(state)` | Average rule (level 1 maximum, thereafter average) across all classes plus CON per level plus feat effects (`hpPerLevel`, `hpFlat`) |
| `calcAC(state)` | Armor class: 10 plus DEX without armor; light/medium/heavy armor with DEX cap per item; shield additive; a magical `bonusAc` on armor counts toward the armor itself (no double counting with `itemBonuses`); unequipped items don't count |
| `itemBonuses(state)` | Flat bonuses `{ac, save, spellDc, spellAtk}` of equipped non-armor items (e.g. Ring of Protection); paused in wild shape |
| `featEffects(state)` | Summed feat effects `{speed, initiative, initiativeProf, hpPerLevel, hpFlat, carryFactor}` (edition-aware via `repo.findFeat`) |
| `effectiveSpeed(state)` / `effectiveInitiative(state)` | Speed or initiative incl. feat effects |
| `carryCapacity(state)` | Carrying capacity `STR * 15`, times feat factor |
| `weaponAttack(state, libItem)` | `{atkBonus, dmgMod, ranged}`: ranged uses DEX, finesse the maximum of STR/DEX, otherwise STR; plus proficiency bonus and fixed magical bonuses of the item |
| `calcSpellSlots(classes)` | `{slots[9], pact, casterLevel}` per the multiclass table. Rounding: a single class matches its own class table (half casters rounded up from level 2, third casters from 3), only the multiclass rule rounds down. Warlock yields `pact = {count, level, arcanum}` with pact slots (1/2/3/4 slots, level up to 5) and Mystic Arcana (levels 6/7/8/9 from levels 11/13/15/17) |
| `primarySpellAbility(classes)` | Spellcasting ability of the first spellcasting class |
| `hitDiceSummary(classes)` | Text like `3d10 + 2d6` |

### dice.js

| Function | Description |
|---|---|
| `roll(sides, count)` | Array of individual rolls; `sides < 1` yields zeros |
| `d20(mod, mode)` | `{raw, first, second, total, isCrit, isFumble}`; `mode` `'adv'`/`'dis'`/`'normal'` |
| `parseAndRoll(formula)` | Parser for `NdS`, `khX`/`klX` (keep highest/lowest), `+`/`-` bonuses; invalid yields `null` |
| `rollFormula(formula)` | Like `parseAndRoll`, additionally publishes `ROLL_RESULT` |
| `rollAbility/rollSkill/rollSave/rollInitiative(...)` | Convenience rolls with correct bonuses (saves incl. item bonuses), publish `ROLL_RESULT` |
| `describeParts(parts)` | Readable roll description |

### bonuses.js

| Function | Description |
|---|---|
| `raceEntry(name)` / `bgEntry(name)` | Raw data entry from the repo |
| `combinedBonus(entry, choice)` | Fixed bonuses plus chosen variant; `choice = {variant, picks}` with `picks` as index-to-ability per weight; duplicate lock prevents duplicate abilities |
| `raceBonusFor(name, choice)` / `bgBonusFor(name, choice)` | Convenient wrappers around `combinedBonus` |
| `featBonusFor(featNames)` | Summed fixed ability bonuses of all feats; duplicates (repeatable feats) count multiple times |

### progression.js and wildshape.js

`asiLevels(className)` returns the ASI levels (Fighter additionally 6/14, Rogue 10), `subclassEntryLevel(className, ruleset)` the subclass entry level (2014: Cleric/Sorcerer/Warlock 1, Druid/Wizard 2, otherwise 3; 2024: uniformly 3), `ALIGNMENTS` the nine alignments.

`wildshapeLimits(classes)` returns `{level, moon, maxCR, noFly, noSwim}` or `null` below level 2 (CR 1/4 from 2, 1/2 from 4, 1 from 8; Circle of the Moon CR 1 or level/3 from 6; swimming from 4, flying from 8). `availableForms(classes)` filters the repo's beast forms accordingly, `elementalUnlocked(classes)` unlocks elementals for Moon Druids from 10, `crToNumber('1/4')` and `formatSpeed(speed)` are helpers, `druidLevel`/`isMoonDruid` read the class list.

## Components (`src/components/`)

Each panel exports a `mount...` function, renders into its root element, and subscribes to `CHAR_CHANGED` with its own `rel` list. Important panels: `CorePanel` (identity, abilities with bonus breakdown, race/background incl. choice dialog, feats with library), `ClassesPanel` (classes, subclasses with automatic spell assignment, level-up dialog with HP choice, ASI or feat and spell selection), `CombatPanel` (HP, AC, attacks, rests, death saving throws), `SpellsPanel` (statistics, slots incl. pact and arcana, known spells, library with class, level, school filters), `InventoryPanel`, `WildshapePanel`, `DicePanel`, `GeneratorPanel` (rules-compliant random character), `DescriptionPanel` (sheet page 2 incl. portrait and comboboxes), `SettingsPanel`, `CharacterSelect` (roster and ruleset selection).

## Data Pipeline (`tools/updater.js`)

Sources: `5etools-mirror-3/5etools-src` (rules) and `5etools-mirror-3/5etools-img` (beast images). Key normalizations:

* Classes: both editions stay in the pack (dedupe by name and source), likewise subclasses (copy with `additionalSpells` preferred). Fields: `spellcasting`, `spellAbility`, `casterProgression`, `saves`, `skillChoices`, `hitDie`.
* Subclass spells: `prepared`/`known` with class-level keys becomes `auto` (automatic assignment), but only with exactly one unnamed variant; named variants (Circle of the Land) and `expanded` (Warlock) only end up in `extra` (selectable library).
* Feats: `parseFeatEffects` reads mechanical bonuses from the rules text (speed only at the start of a sentence, so that conditional bonuses like Charger 2024 don't incorrectly become permanent; initiative fixed or as a proficiency-bonus flag; HP fixed and per level in both phrasings). `parseFeatSpells` reads `additionalSpells` with names and `choose` expressions (`level=2|school=E;N`). `repeatable` marks feats that can be taken multiple times.
* Races and backgrounds: fixed bonuses separated from `abilityChoose` variants (classic and weighted 2024 format unified).
* Spells: school, class lists, damage and healing formulas, saving throw type.

## PDF Export (`utils/pdfExport.js`)

Fills the official sheet via pdf-lib. Worth knowing: the sheet's field names are partly inconsistent (leading/double spaces) and verified via field enumeration. `NeedAppearances` is set so that strict viewers fill the modifier ovals; because viewers thereby regenerate button appearances, the portrait is not set into the image field but drawn directly into the page content (position read at runtime from the "CHARACTER IMAGE" field, fitted preserving the aspect ratio). For tests in Node, the minified pdf-lib does not accept Buffers, only real Uint8Arrays.

## Tests

`node tools/test.mjs` runs the functional test suite: 125 assertions across all exported functions of the core and rules modules, against the real data packs, with stubbed browser globals. Included are, among others, the AC matrix, HP formulas with feats, slot progressions of all caster types incl. pact and arcana, rests per ruleset, edition dedupe with source fallback, homebrew binding, dice statistics, and an i18n coverage scan that checks every text key used in the components in both languages. PDF changes are additionally verified via pypdfium2 rendering (`init_forms()` and `may_draw_forms=True`).

## Typical Extensions

New character field: default in `blankCharacter()`, input in the panel (text inputs with `quietUpdate`), field in the panel's `rel` list, PDF mapping and i18n if needed. New data type from 5etools: fetch and normalization in the updater, getter in the repository (with `bySource` and, for edition collisions, `dedupeByName`), panel or library. New feat effect: pattern in `parseFeatEffects` (watch out for conditional phrasings), summation in `featEffects()`, application at the target location, corpus scan and test case in `tools/test.mjs`.

## Known Limitations

Feat effects are recognized text-based and deliberately applied conservatively; conditional bonuses (e.g. AC only with armor worn) are not applied automatically. The generator rolls randomly within the rules and does not replace optimized builds. The EXE requires a Node.js installation in the PATH.
